### 田中さん単独回｜合計13-18秒

```
【0-10秒】ハイライトモンタージュ
  ├ 田中さん自身の核心的な発言を2-3カット
  └ BGM：同上

【タイトル後】田中さんテーマ提示
```

- ロゴ/タイトルカード表示が5秒以上（離脱ポイントになる）

- 意図のない無音スタート（BGMの敷き忘れ）
- ハイライトなしでいきなりタイトルカード（引きが弱い）

## 2. テロップ仕様

### 2-1. フルテロップ（通常字幕）

全発言を文字起こし表示する。

| 項目 | 仕様 |
|---|---|
| フォント | ゴシック体（ヒラギノ角ゴ / Noto Sans JP） |
| ウェイト | **Medium（500）**（細めで上品に。Boldは使わない） |
| サイズ | **52px固定**（スマホで読めるサイズ／全動画統一） |
| 色 | ウォームホワイト（#FFFDF9） |
| 縁取り | 黒 2px（視認性確保） |
| 影 | ドロップシャドウ 軽め（rgba(0,0,0,0.5)） |
| 位置 | 画面下中央（セーフゾーン内） |
| 行数 | **1行のみ**（2行表示禁止。長い文は分割して順次表示する） |
| 1文の文字数 | **理想12〜16字／上限20字**（1行で必ず収まる長さ）。超える場合は意味の切れ目（句読点・助詞「が」「けれど」「ので」等の後）で分割して順次表示する |
| 表示タイミング | 発言と同期（0.1秒以内のズレに収める） |

### 2-2. 強調テロップ

重要な発言・キーワードをハイライト表示。1動画あたり5-10箇所が目安。

| 項目 | 仕様 |
|---|---|
| フォント | 明朝体（ヒラギノ明朝 / Noto Serif JP） |
| サイズ | **70px固定**（フル52pxと1:1.35比で階層を明確化／全動画統一） |
| 色 | ウォームホワイト(#FFFDF9)（通常） or ゴールド(#C5A059)（特に重要な場面） |
| 背景 | 半透明ディープモーブ帯（rgba(43,40,80,0.85)） |
| 位置 | 画面中央 or やや上 or 画面下中央 ※OPのときは画面中央やや上で使用 |
| 配置の原則 | **人物の顔（話者・ゲスト）に被らない位置を選ぶ**。話者がバストアップ構図のときは画面下中央を避け、顔周りが空く位置（画面中央やや上）を優先する。画面下中央は主にフルテロップ位置と共用なので、対談シーン中の強調表示では特に人物位置を確認する |
| 行数 | **最大2行（3行以上禁止）**。改行は句読点（、。）の直後で行う。意味の切れ目を意識し、文の途中で不自然に切らない |
| アニメーション | フェードイン（0.3秒）→ 2-3秒表示 → フェードアウト |

**強調テロップを入れるタイミング**：
- ゲストの核心的な発言
- 数字・データが出た瞬間
- 視聴者が「これ覚えておきたい」と思う一文
- セカプラキーフレーズ（「整える」「自分で選べる状態をつくる」等）が出た瞬間

**重要：強調テロップ表示中はフルテロップを非表示にする**。両方同時に画面に出すと視認性が下がり、品格も損なわれる。強調テロップが消えた後にフルテロップを再開する。

### 2-3. ネームバー（人物紹介テロップ）

| 項目 | 仕様 |
|---|---|
| 表示タイミング | ゲスト初登場時＋＊シーンの切り替えなどある場合は、動画中盤で1回再表示 |
| 構成 | 名前（大）＋ 肩書（小）の2段 |
| 名前フォント | **明朝体**（ヒラギノ明朝 ProN / Noto Serif JP）※人名は格式領域のため明朝 |
| 肩書フォント | ゴシック体（ヒラギノ角ゴ ProN / Noto Sans JP）※補足情報＝BODY相当 |
| 名前色 | ウォームホワイト(#FFFDF9) |
| 肩書色 | ゴールド(#C5A059) |
| 背景 | 半透明ディープモーブ（角丸8px） |
| 位置 | 画面左下 |
| 表示時間 | 4-5秒 |
| アニメーション | 左からスライドイン → スライドアウト |

### 2-4. 数字表記ルール

テロップ内の数字は**半角アラビア数字を基本**とし、慣用表現のみ漢数字を使う。**全角数字は原則使わない**。

**基本方針の根拠**：
- 50-60代の老眼＋スマホ視聴では半角アラビア数字が最も瞬時認識しやすい
- 金融・健康・資産の具体値は数字の鮮度が信頼の源（「三千円」より「3,000円」）
- 横書きテロップでは半角が現代的な上品さ。全角は可読性も格式も中途半端

#### 場面別の使い分け

| 場面 | 表記 | 例 |
|---|---|---|
| 金額 | 半角＋3桁区切り＋漢字単位 | `月3,000円` / `120万円` / `5,000万円` |
| 年代・年齢 | 半角 | `50代` / `62歳` / `50-60代` |
| 年数・期間 | 半角 | `10年後` / `3ヶ月` / `週2本` |
| 時間・分数 | 半角 | `15分` / `20-25分` |
| 割合・% | 半角 | `70%` / `20-25%` |
| 順位・番号 | 半角 | `第1位` / `5つの資産` / `3つのポイント` |
| 人数 | 半角 | `顧客3名` / `1,000人` |
| 日付 | 半角 | `2026年4月` / `4月15日` |
| 熟語・慣用表現 | **漢数字** | `一歩ずつ` / `二度と` / `十人十色` / `百年人生` |
| 詩的・概念的表現 | **漢数字** | `一つひとつ整える` / `一生の習慣` |
| ことわざ・格言 | **漢数字** | `千里の道も一歩から` |

#### 判断基準：「数えているか、語っているか」

- 数えている（量・具体値）→ 半角：`1年で120万円増`
- 語っている（概念・情緒）→ 漢数字：`一年発起` / `一つひとつ整える`

同じ「1」でも文脈で使い分ける。

#### ゴールド強調との組み合わせ

数字が発言の核心になる場面では、**数字部分のみゴールド(#C5A059)＋明朝で強調**してよい。**ただし強調テロップ限定**。フルテロップでは色変えしない（読み流しを妨げるため）。

例：「月 **3,000円** の見直しで、12ヶ月で **120万円** 増」（太字部分をゴールド）

### 2-5. 用語解説テロップ

専門用語・固有名詞が登場した際、50〜60代の視聴者が意味を理解できるよう短い解説を表示する。本編中の個別表示（2-5-1）と動画末尾の一覧再掲（2-5-2）の2形式で運用する。

#### 2-5-1. 個別表示（本編中）

強調テロップ仕様（§2-2）に準拠。

| 項目 | 仕様 |
|---|---|
| 用途 | 専門用語・業界用語・年代差のある固有名詞が出た瞬間 |
| 表示形式 | 「用語」＋「解説文」の2要素 |
| 解説文の長さ | 15〜20字（1行で収まる長さ） |
| 用語色 | アンティークゴールド(#C5A059) |
| 解説文色 | ウォームホワイト(#FFFDF9) |
| その他仕様 | §2-2強調テロップに準拠 |

**選定基準**：50〜60代女性ペルソナが「聞き覚えはあるが正確な意味を説明できない」用語。金融用語・業界隠語・年代固有の固有名詞（例：電電公社、8トラ等）が代表例。

**解説文のトーンNG**：
- 「儲かる」「ボロボロ」等のカジュアル表現
- 由来・語源の断定（諸説ある場合は避ける）
- ※注記はテロップ内に入れない（免責は§12で別途表示）

#### 2-5-2. 用語集の末尾再掲（エンドカード）

本編で表示した用語解説を、動画末尾のチャンネル登録CTAの直前に一覧で再掲する。50〜60代視聴者の理解定着・記憶を助ける目的。

| 項目 | 仕様 |
|---|---|
| 表示タイミング | 動画末尾、チャンネル登録CTAの直前（固定） |
| 表示時間 | 5〜7秒（一時停止して読めるよう余裕をもたせる） |
| 構成 | 「本編に出てきた用語集」見出し＋全用語＋短い解説を一画面に |
| 用語数の上限 | 1画面8個まで（超える場合は2画面に分ける） |
| 解説文の長さ | 1用語あたり15〜20字（§2-5-1に準拠） |
| デザイン | §2-5-1の色・書体に準拠（明朝＋アンティークゴールド＋ウォームホワイト） |

### テロップのNG

- 赤・黄色の強調色（品格毀損）
- 絵文字・スタンプの多用
- バラエティ風の効果音付きテロップ
- フルテロップの2行表示（長い文は分割して1行ずつ順次表示）
- フルテロップ1文が20字を超える表示（読み切る前に切り替わる／品格毀損）
- フルテロップのBold（700）使用（Medium(500)で統一）
- 強調テロップの3行以上表示（最大2行。収まらない場合は文を短くする）
- 強調テロップの不自然な改行（句読点の直後で改行する。文の途中で切らない）
- 強調テロップとフルテロップの同時表示（強調表示中はフルテロップを消す）
- 文字が小さすぎる（40px未満）
- 画面上に3箇所以上のテロップ同時表示
- 全角数字の使用（`３，０００円`）／半角と全角の混在（`3,０００円`）
- 桁区切りなしの大きな数字（`120000円` → `12万円` or `120,000円`）
- 熟語の算用数字化（`3日坊主` → `三日坊主`）

## 3. BGM・音声設計

### 3-1. 使用シーンと選定基準

| シーン | トーン | 音量目安 | 例 |
|---|---|---|---|
| OPジングル（2-3秒） | 品のあるピアノ単音 or 弦楽 | 発言の80% | タイトルカード表示時のみ |
| 本編・対談中 | 静かなアンビエント / ソフトピアノ | **発言の10-15%**（存在感消す） | 途切れず流す。邪魔しない |
| 話題の転換点 | 軽いピアノフレーズ | 発言の30% | テーマ切り替わり時に短く |
| エンディング（CTA時） | 冒頭ジングルと呼応するピアノ | 発言の50% | CTA口頭案内と同時にフェードイン |
| 感動・共感シーン | 弦楽 or アコースティックギター | 発言の20% | ゲストの体験談で使用 |

### 3-2. BGM選定のルール

**推奨ジャンル**：
- クラシカル・ピアノ
- アコースティック（ギター・弦楽）
- ジャズ（落ち着いたもの）
- アンビエント（環境音楽系）

**NG**：
- EDM / ポップ / ヒップホップ
- テンション高い曲（煽り感が出る）
- 歌詞ありの曲（発言と干渉）
- 暗い・重い曲（ドキュメンタリー風にしすぎない）

**音源候補**：
- YouTubeオーディオライブラリ（無料）
- Artlist / Epidemic Sound（有料・品質高）
- CapCut内蔵BGM（手軽だが品質に注意）

### 3-3. BGM運用の注意

- **本編BGMは1曲をループ**で通す（曲変えすぎると散漫になる）
- **冒頭ジングルとエンディングは同じ曲の変奏**にすると統一感が出る
- **意図的な無音は演出として有効**：ゲストの核心的発言の直前にBGMをフェードアウト→発言が際立つ。ただしBGMの「敷き忘れ」とは区別する
- BGMの切り替えは**フェードイン/アウト（1-2秒）**で滑らかに

## 4. 編集トーン

### 4-1. カット割り

| 項目 | 仕様 |
|---|---|
| 基本テンポ | ゆったり。1カット3-5秒以上 |
| ジャンプカット | 最小限（呼吸・間を残す。バラエティ風に詰めない） |
| カメラ切り替え | 話者が変わるタイミングで自然に切り替え |
| インサート映像 | 話題に関連する写真・図を適宜挿入（5-10秒） |

### 4-2. SE（効果音）

| 使ってよいSE | 使ってはいけないSE |
|---|---|
| 場面転換時の軽いチャイム | ドドン！系の衝撃音 |
| テーマ表示時のソフトな音 | ピンポン！正解音 |
| — | ブブー！不正解音 |
| — | 笑い声・拍手SE |

**原則：SEは極力使わない。**使う場合も1動画あたり3-5箇所まで。

### 4-3. ズーム・エフェクト

| 項目 | 仕様 |
|---|---|
| ズーム | ゲストの重要発言時にゆっくり10-15%ズームイン（0.5秒かけて） |
| パン（横移動） | インサート映像・風景カットなどで使用可 |
| ティルト（縦移動） | インサート映像・空間の全体像を見せる場面で使用可 |
| トランジション | カット or クロスフェード（0.3秒）のみ。スライド・回転禁止 |
| ぼかし | 撮影時に設定が基本。ただし背景整理やフォーカス演出のために**編集時の追加も可** |

### 4-4. カラーグレーディング（色補正）

撮影環境による映像のばらつきを抑え、チャンネル全体の統一感を保つために、全動画に共通のカラーグレーディングを適用する。

| 項目 | 仕様 |
|---|---|
| 色温度 | **やや暖色寄り（5500-6000K目安）** → セカプラの「温かみ」トーンと整合 |
| コントラスト | 控えめ（フラットすぎず、強すぎず） |
| 彩度 | 自然〜やや低め（派手にしない） |
| ハイライト | 飛ばしすぎない（白飛び防止） |
| 肌色 | 健康的で自然に見える補正（レタッチ過剰NG） |
| 参考トーン | NoBorder・鎌倉市動画の暖色寄りグレーディング |

**運用ルール**：
- 初回撮影時にグレーディングのプリセットを作成し、以降の動画に同じプリセットを適用
- CapCutの「フィルター」ではなく「調整」機能で手動設定（フィルターは色味が過剰になりやすい）
- ゲストが変わっても、色温度・コントラストの基準値は固定

## 5. 視聴維持率を高める構成設計（YouTube成長施策）

YouTubeアルゴリズムが最も重視する**平均視聴維持率**を高めるための編集設計。

### 5-1. パターンインタラプト（3-5分ごと）

視聴者の飽きを防ぐため、3-5分ごとに**視覚的な変化**を入れる。

| 手法 | 頻度 | 例 |
|---|---|---|
| 強調テロップ表示 | 3分ごと | 核心発言をディープモーブ帯＋明朝体で表示 |
| インサート映像 | 5分ごと | 話題に関連する図表・写真を5-10秒挿入 |
| BGM変化 | 話題の転換時 | フェードアウト→新しいフレーズ |
| カメラアングル変更 | 随時 | 寄り↔引きの切り替え |

### 5-2. フック構造（冒頭〜30秒）※§1のOP構成と連動

| 秒数 | やるべきこと | YouTubeアルゴリズム上の意味 |
|---|---|---|
| 0-10〜15秒 | ハイライトモンタージュ（3-5カット＋BGM） | **世界観を伝え「見たい」を育てる** |
| +3秒 | タイトルカード | ブランド認知 |

### 5-3. エンゲージメント促進（コメント・いいね）

**推奨タイミング**：動画の中盤（10-15分地点）

### 5-4. カード（mid-roll リンク）

- 「詳しくはこちらの動画で話しています」と口頭で言及

| 尺 | アルゴリズム上の特性 |

### 12-1-3. 画面内表示の判断基準

概要欄常設（§12-2）だけでは不十分で、**画面内にも免責を表示すべきセンシティブ箇所**：

1. **投資の具体的なリターンが提示される箇所**（例：年利◯%、◯年で◯倍等）
2. **特定の健康法・療法の効果が断定的に語られる箇所**（医療効果の示唆を含む）
3. **特定の金融商品・銘柄・事業者への誘導に聞こえる箇所**

これらはYouTubeショート切り抜き時に免責が外れるリスクがあるため、画面内表示（§12-1-2）で二重にカバーする。

**画面内表示は最小限に**：動画冒頭の一括表示ではなく、該当シーン冒頭のみピンポイント配置する（冒頭一括は離脱要因になり、§1のOP構成にも抵触するため）。

### 12-3. 関連法規への注意事項

| 法規 | セカプラYouTubeで注意すべき点 |
|---|---|
| **金融商品取引法** | 特定の銘柄・金融商品の推奨は投資助言業の登録が必要。「〇〇を買うべき」「この商品がおすすめ」は絶対NG。「こういう考え方がある」「専門家に相談を」に留める |
